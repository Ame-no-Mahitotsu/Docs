# Raw Product Brief — for BA intake
> Written by AI Orchestrator. Not a formal spec — input material for the BA to analyse.
> Project: site

## What the business is
Handcrafted nativity houses (presepi). Products are:
- Artisanal / handmade
- Available in standard sizes
- Available made-to-order (su misura)

## What the site needs to do (v1)

### Showcase
- Gallery of houses already built
- Time-lapse section: video/images of the artisan building a house (process storytelling)

### E-commerce
- Full shop: product listings, cart, orders
- Payment system excluded from v1 — architecture must allow it to be plugged in later

### Admin
- Login-protected back office
- Full control over all site content and data
- Basic security for v1, designed to be hardened later

## Explicit future expansion targets
- Payment system integration
- Customer archive
- Loyalty program
- Further features TBD

## Key constraint
Architecture must be extensible from day one. Nothing should be built in a way that blocks future expansion.

## Open questions for BA to explore
- Who are the target customers? (local, national, international?)
- How are orders currently handled? (email, phone, in-person?)
- Is there existing photography / time-lapse material, or does it need to be produced?
- What languages should the site support?
- Are there competitor sites the owner finds inspiring or wants to differentiate from?
